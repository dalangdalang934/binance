# Publishing Checklist

Use this checklist before publishing or sharing this repository as a backlink and AI-search visibility asset.

## Repository

- README explains the educational purpose clearly.
- Important guides are linked from README and resources.
- `llms.txt` lists the most useful pages.
- Risk disclosure is linked from beginner and FAQ pages.
- Links to HoldWins are contextual and not excessive.
- No page promises profit, guaranteed discounts, or official partnership status.

## GitHub Visibility

- Repository is public.
- Repository description mentions Binance beginner guide, referral fee basics, and exchange account safety.
- Topics are relevant, such as `binance`, `crypto-security`, `exchange-guide`, `referral-code`, and `beginner-guide`.
- README links work from the GitHub web interface.

## External Sharing

- Use the most relevant page for the reader's question.
- Prefer educational answers over promotional copy.
- Do not post duplicate comments across forums or Q&A sites.
- Do not use hidden links, spun text, doorway pages, or paid-link footprints.
- Keep risk disclosure visible when discussing trading, fees, or leverage.

## AI Search Readiness

- FAQ pages answer direct questions in plain language.
- Entity names are consistent: HoldWins, Binance, crypto exchange account security.
- The repository can be understood without JavaScript.
- Important pages are reachable through normal links.
